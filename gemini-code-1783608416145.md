# SPEC-PHASE-2.md — Update "Foto Kita Blur" (Photo Booth Expansion)

> Dokumen ini merupakan lanjutan dari Spec Fase 1 (Video Recorder), menambahkan fungsionalitas pengambilan foto statis bertema kolase/grid dengan tetap memanfaatkan AI hand-tracking lokal.

---

## 1. Overview Fase 2

**Nama project:** Foto Kita Blur — Photo Booth Expansion
**Tujuan:** Mengekspansi aplikasi web dari sekadar perekam video menjadi pengalaman interaktif ganda (Video Recorder dan Photo Booth). Konsep efek blur dinamis yang dipicu secara otomatis oleh deteksi gesture (misalnya peace sign) melalui MediaPipe Hands di sisi client akan dipertahankan dan diadaptasi untuk pengambilan foto statis berformat grid (kolase) layaknya mesin photo box fisik.

---

## 2. User Stories (Tambahan Fase 2)

1. **Sebagai user**, saya ingin memiliki pilihan mode di awal sebelum mulai: merekam video secara real-time atau mengambil foto bergaya photo booth.
2. **Sebagai user**, pada mode foto, saya ingin bisa memilih layout grid (misal: 1, 2, 3, atau 4 foto bersusun) layaknya di mesin photo box.
3. **Sebagai user**, saat mode foto berjalan, saya ingin efek blur tetap bisa muncul ketika saya melakukan gesture target (misal peace sign) sebelum kamera mengambil jepretan statis.
4. **Sebagai user**, saya ingin hasil foto saya memiliki bingkai (frame) estetis layaknya polaroid dengan border putih yang bisa langsung saya unduh ke perangkat saya.

---

## 3. Functional Requirements (FR) Fase 2

| ID | Requirement | Keterangan |
|---|---|---|
| FR-13 | Sistem menampilkan opsi pemilihan mode "Rekam Video" atau "Photo Booth" di UI utama. | Mode video tetap mempertahankan fitur `MediaRecorder` API dari Fase 1. |
| FR-14 | Pada mode "Photo Booth", sistem menyediakan opsi layout grid foto (misalnya 1x1, 1x2, 1x3, 1x4). | Menyesuaikan jumlah frame jepretan. |
| FR-15 | Sistem menerapkan hitungan mundur (countdown) yang berulang otomatis (looping) sesuai dengan jumlah bingkai pada layout yang dipilih user. | Modifikasi fungsi hitungan mundur 3 detik menjadi berulang per jepretan. |
| FR-16 | Sistem menangkap (jepret) frame dari HTML5 `<canvas>` tepat saat hitungan mundur mencapai nol, lalu menyimpannya ke dalam layout grid sementara. | Trigger pengambilan gambar statis. |
| FR-17 | Sistem tetap menyediakan tombol trigger manual (keyboard Spasi atau tombol layar) sebagai cadangan untuk mengaktifkan status blur pada mode foto. | Fallback jika tracking tangan gagal. |
| FR-18 | Sistem memproses hasil akhir kolase foto dan memungkinkan user mengunduhnya dalam format gambar statis (misal `.png` atau `.jpeg`). | Output akhir mode foto. |

## 4. Technical Notes — Update Implementasi

Pendekatan teknis untuk mengintegrasikan fitur foto ke dalam arsitektur yang sudah ada:

* **Manajemen State Canvas:** Pada mode Fase 2 (mode foto), aliran `MediaRecorder` dihentikan. Sebagai gantinya, sistem akan mengeksekusi perintah `canvas.toDataURL()` tepat pada detik ke-0 countdown untuk menangkap gambar statis.
* **Penggabungan Grid (Hidden Canvas):** Sistem menggunakan satu `<canvas>` tambahan yang disembunyikan (hidden) di latar belakang. Canvas ini berfungsi khusus untuk menempelkan hasil-hasil `toDataURL()` sebelumnya menjadi satu kesatuan (kolase) dan menambahkan elemen visual seperti garis tepi (border putih).
* **Konsistensi Efek Visual:** Efek soft blur, film grain halus, dan warna hangat tetap diaplikasikan langsung pada manipulasi pixel canvas utama. Intensitas efek (Ringan / Sedang / Brutal) tetap diterapkan pada hasil jepretan foto statis.
* **Keamanan Privasi:** Proses penggabungan layout foto statis dan hand-tracking diproses sepenuhnya di browser (client-side) tanpa upload data ke server.

## 5. Acceptance Criteria (Fase 2)

- [ ] Sistem berhasil menampilkan antarmuka pemilihan mode (Video vs. Photo Booth) yang intuitif di awal sesi.
- [ ] Sistem berhasil melakukan hitungan mundur berulang sesuai layout foto yang dipilih.
- [ ] State efek blur berhasil dipicu oleh gesture target atau tombol manual sesaat sebelum jepretan statis diambil pada mode photo booth.
- [ ] Hasil unduhan foto berupa satu file gambar statis yang menggabungkan seluruh jepretan.
- [ ] Hasil unduhan foto memiliki elemen visual estetis (vignette, film grain, bingkai) yang di-render di client-side.
- [ ] Fitur rekam video dari Fase 1 tetap berjalan normal tanpa gangguan dan masih dapat menghasilkan file `.webm`.
